## 一、技术栈

### 核心技术选型

| 维度 | 选型 | 版本 | 理由 |
|---|---|---|---|
| 语言 | TypeScript | ^5.0 | Obsidian 插件生态标准，强类型 |
| 运行环境 | Obsidian Plugin API | latest | 目标宿主 |
| UI | 原生 TypeScript + DOM | — | 零依赖、轻量；无复杂 state UI 需求 |
| 打包 | esbuild | 官方模板版本 | 配置简单，Obsidian 官方模板默认 |
| 复习算法 | ts-fsrs | ^4.x | MIT 许可，FSRS-6 实现，原生 TS |
| 哈希 | 内置 Web Crypto API (SubtleCrypto) | — | 零依赖，用于笔记内容 hash |
| diff（可选） | 自实现简单长度+hash 对比 | — | MVP 不用 diff 库，后续视需要引入 |

### 开发依赖

```json
{
  "devDependencies": {
    "@types/node": "...",
    "obsidian": "latest",
    "typescript": "^5.0",
    "esbuild": "...",
    "vitest": "..."  // 纯函数测试

}
```

### 不用的技术（显式说明）

- ❌ React / Vue / Svelte：UI 复杂度不需要
- ❌ Redux / MobX / Zustand：状态简单，单例 state manager 足够
- ❌ Lodash / Ramda：标准库够用
- ❌ diff-match-patch：MVP 阶段对 diff 的需求仅限"是否变化 + 变化量"，不需要真正的 diff

## 二、模块划分

### 整体架构

```
┌─────────────────────────────────────────────────────────┐
│                    ResurfacePlugin                      │
│                    (main entry, 生命周期)                 │
└───────┬─────────────────────────────────────────┬───────┘
        │                                         │
        ▼                                         ▼
┌─────────────────┐                       ┌──────────────────┐
│   ObsidianLayer │                       │    UILayer       │
│   (事件/IO)     │                       │    (View/渲染)    │
├─────────────────┤                       ├──────────────────┤
│ - VaultWatcher  │                       │ - SideBarView    │
│ - NoteIO        │                       │ - ReviewTab      │
│ - RibbonBadge   │                       │ - NoticeRenderer │
│ - StartupNotice │                       │ - SettingsTab    │
└────────┬────────┘                       └────────┬─────────┘
         │                                         │
         │         ┌─────────────────────┐         │
         └────────▶│    DomainLayer      │◀────────┘
                   │    (业务核心)        │
                   ├─────────────────────┤
                   │ - StorageService    │  ← data.json
                   │ - FSRSService       │  ← ts-fsrs 封装
                   │ - Scheduler         │  ← 今日列表/排序
                   │ - EditTracker       │  ← 编辑 diff/阈值
                   │ - StatsService      │  ← 统计指标
                   │ - CueExtractor      │  ← TLDR 提取
                   └─────────────────────┘
```

### 每个模块的职责

#### `ResurfacePlugin`（主入口）
- 继承 Obsidian `Plugin`
- 加载/卸载插件
- 注册所有事件、命令、View、SettingsTab
- 初始化 DomainLayer 所有服务

#### `ObsidianLayer`（Obsidian 集成）

**VaultWatcher**
- 监听 `vault.on('modify')` → 触发 EditTracker
- 监听 `vault.on('rename')` → 更新 StorageService 的 path key
- 监听 `vault.on('delete')` → 归档或移除 data

**NoteIO**
- 读取笔记内容（`vault.read`）
- 提供给 CueExtractor 和 EditTracker 使用

**RibbonBadge**
- 侧栏 ribbon 的创建和数字更新
- 点击处理：打开侧栏 View

**StartupNotice**
- 插件启动时的顶部通知
- 只在有待复习时显示

#### `UILayer`（界面）

**SideBarView**（`ItemView`）
- 注册为 `VIEW_TYPE_REVIEW_SIDEBAR`
- 默认位置：右侧栏
- 渲染：cue / 评分按钮 / 统计 / 空状态
- 订阅 Scheduler 的状态变化

**ReviewTab**
- 复习专用 tab，显示当前笔记内容
- 按需创建：SideBarView 的"展开正文"按钮触发
- 用户关闭后再次"展开"时重新创建

**NoticeRenderer**
- 顶部通知的渲染逻辑
- 文案按产品文档的调性规范

**SettingsTab**（`PluginSettingTab`）
- 基础设置 + 高级设置折叠区
- 14 项设置按产品文档定义

#### `DomainLayer`（领域层）

**StorageService**
- 封装 `this.app.loadData()` / `this.app.saveData()`
- 提供类型安全的读写 API
- 处理 data.json 的迁移版本

**FSRSService**
- 封装 ts-fsrs 调用
- `scheduleNext(note, rating)` → 更新 S/D/R，返回新 next_review
- `calculateRetrievability(note, at)` → 计算当前 R 值
- `adjustStabilityForEdit(note)` → 执行 Stability × 0.5
- 管理用户个性化的 FSRS 参数

**Scheduler**
- 计算"今日列表"：扫描 data.notes，按 R 值排序，取前 N 条
- 日边界处理（凌晨 04:00）
- 筛除逻辑：excluded、已完成今日复习

**EditTracker**
- 监听 modify 事件（通过 VaultWatcher）
- Debounce 5 秒
- 计算 diff：长度变化 + hash 对比
- 触发阈值判断 → 调用 FSRSService 调整 S

**StatsService**
- 累计复习次数、streak 天数、复活池大小
- 从 revlog 聚合
- 日边界更新 streak

**CueExtractor**
- 按 fallback 优先级提取 TLDR
- 输入：笔记路径
- 输出：标题 + TLDR 字符串

### 依赖方向（严格遵守）

```
Obsidian API
    ↑
ObsidianLayer / UILayer
    ↑
DomainLayer
```

- **DomainLayer 不依赖 Obsidian API**（除了类型）
- **UILayer 不直接读写 Obsidian 文件**
- **DomainLayer 可以被纯函数测试**（这是选择"只测纯函数"的基础）

## 三、核心数据流

### 数据流 1：应用启动

```
ResurfacePlugin.onload()
  ├─ StorageService.load() ← data.json
  ├─ FSRSService.init(settings)
  ├─ VaultWatcher.register()
  ├─ RibbonBadge.create()
  ├─ 注册 SideBarView 和 SettingsTab
  └─ Scheduler.getTodayList()
       └─ RibbonBadge.setCount(N)
       └─ StartupNotice.showIfNeeded(N)
```

### 数据流 2：用户点击 Ribbon 开始复习

```
用户点击 RibbonBadge
  └─ 打开/激活 SideBarView
       └─ Scheduler.getTodayList() → 第一条
            └─ CueExtractor.extract(note) → 标题 + TLDR
                 └─ 渲染到 SideBarView
                      └─ 主区保持用户原状态
```

### 数据流 3：用户点击"展开正文"

```
SideBarView 的"展开正文"按钮
  ├─ 如果 ReviewTab 不存在 → 创建
  ├─ 如果存在 → 复用
  └─ ReviewTab.showNote(notePath)
       └─ 使用 Obsidian 的 workspace.openLinkText
       └─ SideBarView 切换到"评分"状态
```

### 数据流 4：用户评分（Good）

```
SideBarView 的 Good 按钮
  └─ FSRSService.scheduleNext(note, 'good')
       ├─ 计算新的 S/D/next_review
       ├─ 追加 revlog 条目
       └─ StorageService.save()
  └─ SideBarView 切换到"等待下一条"状态
  └─ 主区停留在当前笔记（用户可编辑）
```

### 数据流 5：用户点击"下一条"

```
SideBarView 的"下一条"按钮
  └─ Scheduler.getTodayList() → 下一条
       (实时计算，反映最新 data 状态)
  └─ ReviewTab.showNote(nextNotePath)
  └─ SideBarView 切换到 cue 状态
```

### 数据流 6：笔记被编辑（触发阈值）

```
Obsidian 的 modify 事件
  └─ VaultWatcher 转发给 EditTracker
       └─ Debounce 5 秒
            └─ NoteIO.read(path) → 当前内容
            └─ 对比 data.notes[path].lastSnapshot*
                 ├─ 不超阈值 → 仅更新 snapshot，结束
                 └─ 超阈值 →
                      ├─ 检查 lastEditTriggerAt < 7 天前
                      │    └─ 是 → 静默，结束
                      └─ 否 → FSRSService.adjustStabilityForEdit(note)
                           └─ S × 0.5 → 重算 next_review
                           └─ 更新 lastEditTriggerAt
                           └─ StorageService.save()
                           └─ 如果侧栏正在显示，通知 SideBarView 刷新列表
```

### 数据流 7：笔记重命名

```
Obsidian 的 rename 事件 (file, oldPath)
  └─ VaultWatcher 转发给 StorageService
       └─ data.notes[newPath] = data.notes[oldPath]
       └─ delete data.notes[oldPath]
       └─ revlog 中 path 字段批量更新
       └─ StorageService.save()
```

### 数据流 8：笔记删除

```
Obsidian 的 delete 事件 (file)
  └─ StorageService.archiveNote(path)
       └─ 把 data.notes[path] 移到 data.archive[path]
       └─ 保留 revlog（未来可能恢复）
       └─ 不真正删除，用户如果是误删可恢复
```

## 四、数据模型

### data.json 完整 Schema

```typescript
interface ResurfaceData {
  version: number;                      // schema 版本，用于迁移
  settings: Settings;
  notes: Record<NotePath, NoteState>;
  archive: Record<NotePath, NoteState>; // 被删除/排除的笔记
  revlog: ReviewLogEntry[];
  stats: Stats;
  fsrsParams?: FSRSParameters;          // 用户个性化参数，未训练时为空
}

type NotePath = string;  // 相对 vault 根的路径

interface Settings {
  // 基础
  dailyLimit: number;              // 默认 15
  firstReviewDays: number;         // 默认 3
  firstReviewJitter: number;       // 默认 1
  ratingMode: '2-button' | '4-button';  // 默认 '2-button'
  autoAdvance: boolean;            // 默认 false

  // 高级
  desiredRetention: number;        // 默认 0.9
  tldrFieldName: string;           // 默认 'tldr'
  tldrFallbackLength: number;      // 默认 200
  showStreak: boolean;             // 默认 true

  editThresholdAbsolute: number;   // 默认 50
  editThresholdRatio: number;      // 默认 0.2
  editTriggerAction: 'x0.3' | 'x0.5' | 'x0.7' | 'reset' | 'none';  // 默认 'x0.5'

  dayBoundaryHour: number;         // 默认 4 (凌晨 04:00)
}

interface NoteState {
  // FSRS 核心状态
  stability: number;
  difficulty: number;
  lastReview: string;    // ISO 8601
  nextReview: string;    // ISO 8601
  state: 'new' | 'learning' | 'review' | 'relearning';
  reps: number;          // 复习次数
  lapses: number;        // 忘记次数

  // 排除
  excluded: boolean;
  excludedAt: string | null;

  // 编辑跟踪
  lastSnapshotLength: number;
  lastSnapshotHash: string;
  lastEditTriggerAt: string | null;

  // 入池时间（用于统计"首次复习"的节点）
  addedAt: string;
}

interface ReviewLogEntry {
  path: NotePath;
  timestamp: string;         // ISO 8601
  rating: 1 | 2 | 3 | 4;     // 1=Again, 4=Easy
  elapsedDays: number;       // 距上次复习的天数
  scheduledDays: number;     // 本次计算出的下次间隔
  stabilityBefore: number;
  stabilityAfter: number;
  difficultyAfter: number;
}

interface Stats {
  totalReviews: number;
  streakDays: number;
  lastReviewDate: string;    // ISO date (不含时间)，用于 streak 计算
  firstUseDate: string;
}

interface FSRSParameters {
  w: number[];               // 长度 21 的数组，FSRS-6 参数
  lastOptimizedAt: string;
  trainingSize: number;      // 训练时用了多少条 revlog
}
```

### Schema 迁移策略

- 顶层 `version` 字段
- StorageService 加载时检查 version，执行必要迁移
- 每次 breaking change 写一个 migrator

## 五、Obsidian 集成层

### Plugin 生命周期

```typescript
export default class ResurfacePlugin extends Plugin {
  storage: StorageService;
  fsrs: FSRSService;
  scheduler: Scheduler;
  editTracker: EditTracker;
  // ...

  async onload() {
    // 1. 加载数据
    await this.storage.load();

    // 2. 初始化服务
    this.fsrs = new FSRSService(this.storage);
    this.scheduler = new Scheduler(this.storage, this.fsrs);
    this.editTracker = new EditTracker(this.storage, this.fsrs, this);

    // 3. 注册 Vault 事件
    this.registerEvent(this.app.vault.on('modify', (file) => {
      if (file instanceof TFile && file.extension === 'md') {
        this.editTracker.onModify(file);
      }
    }));
    this.registerEvent(this.app.vault.on('rename', (file, oldPath) => {
      this.storage.onRename(oldPath, file.path);
    }));
    this.registerEvent(this.app.vault.on('delete', (file) => {
      if (file instanceof TFile) this.storage.onDelete(file.path);
    }));
    this.registerEvent(this.app.vault.on('create', (file) => {
      if (file instanceof TFile && file.extension === 'md') {
        this.storage.onCreate(file.path);
      }
    }));

    // 4. 注册 UI
    this.registerView(VIEW_TYPE_SIDEBAR, (leaf) => new SideBarView(leaf, this));
    this.addRibbonIcon('sprout', '打开 Resurface 复习', () => this.activateSideBar());
    this.addSettingTab(new ResurfaceSettingTab(this.app, this));

    // 5. 启动通知
    this.showStartupNoticeIfNeeded();
  }

  async onunload() {
    // 关闭 View
    this.app.workspace.detachLeavesOfType(VIEW_TYPE_SIDEBAR);
    // 保存数据（防止进程结束前的丢失）
    await this.storage.save();
  }
}
```

### 事件监听细节

#### `vault.on('create')`
- 新笔记创建 → `StorageService.onCreate(path)` 初始化 NoteState
- `addedAt = now`，`firstReviewAt = now + (firstReviewDays ± jitter)`
- FSRS 初始 S/D 由 default parameters 给出

#### `vault.on('modify')`
- debounce 5 秒后触发 EditTracker
- 不在 modify 内做重活——Obsidian 频繁保存会影响体验

#### `vault.on('rename')`
- 同步更新 data.notes 的 key
- 批量更新 revlog 里的 path

#### `vault.on('delete')`
- 归档到 data.archive（软删除）
- 保留 revlog（用户可能只是移到 .trash）

### ReviewTab 的管理

```typescript
// 按需创建/复用逻辑
async openReviewTab(notePath: string) {
  let leaf = this.app.workspace.getLeavesOfType(VIEW_TYPE_REVIEW_TAB)[0];
  if (!leaf) {
    leaf = this.app.workspace.getLeaf('tab'); // 主区新 tab
```

但这里有个选择：**ReviewTab 到底是自定义 View 类，还是就用 Obsidian 原生的 MarkdownView 打开笔记？**

**推荐：用 MarkdownView**。理由：
- 笔记的所有编辑/链接/预览能力直接复用
- 用户在复习 tab 里能享受完整的 Obsidian 体验
- 不需要重新实现 md 渲染

那么"复用 tab"的标识怎么做？用一个**特殊 tab 属性**（group 或者自定义 state），在每次复习切换时识别并复用。具体实现时用 `workspace.openLinkText(notePath, '', false, { state: { ... } })` 并打 tag。

## 六、FSRS 集成层

### 封装思路

```typescript
import { FSRS, generatorParameters, Rating, Card, State } from 'ts-fsrs';

class FSRSService {
  private fsrs: FSRS;
  private params: FSRSParameters | null;

  constructor(private storage: StorageService) {
    this.params = storage.data.fsrsParams;
    this.fsrs = new FSRS(generatorParameters({
      w: this.params?.w, // 未训练时用 default
      requestRetention: storage.data.settings.desiredRetention,
      maximumInterval: 36500,  // 100 年
      enableFuzz: true,
    }));
  }

  scheduleNext(note: NoteState, rating: Rating): NoteState {
    const card = this.toCard(note);
    const log = this.fsrs.repeat(card, new Date());
    const result = log[rating];
    return {
      ...note,
      stability: result.card.stability,
      difficulty: result.card.difficulty,
      lastReview: now(),
      nextReview: result.card.due.toISOString(),
      state: result.card.state,
      reps: result.card.reps,
      lapses: result.card.lapses,
    };
  }

  calculateRetrievability(note: NoteState, at: Date = new Date()): number {
    const elapsed = daysBetween(new Date(note.lastReview), at);
    return this.fsrs.forgetting_curve(elapsed, note.stability);
  }

  adjustStabilityForEdit(note: NoteState): NoteState {
    const action = this.storage.data.settings.editTriggerAction;
    let factor = 0.5;
    if (action === 'x0.3') factor = 0.3;
    if (action === 'x0.5') factor = 0.5;
    if (action === 'x0.7') factor = 0.7;
    if (action === 'none') return note;
    if (action === 'reset') {
      return this.resetToNew(note);
    }

    const newStability = note.stability * factor;
    const card = this.toCard({ ...note, stability: newStability });
    // 根据新 S 重算 nextReview
    return {
      ...note,
      stability: newStability,
      nextReview: this.fsrs.nextInterval(newStability, note.state).toISOString(),
    };
  }

  private toCard(note: NoteState): Card {
    return {
      due: new Date(note.nextReview),
      stability: note.stability,
      difficulty: note.difficulty,
      elapsed_days: daysBetween(new Date(note.lastReview), new Date()),
      scheduled_days: daysBetween(new Date(note.lastReview), new Date(note.nextReview)),
      reps: note.reps,
      lapses: note.lapses,
      state: note.state,
      last_review: new Date(note.lastReview),
    };
  }
}
```

### Optimizer（MVP 之后）

FSRS 参数训练不是 ts-fsrs 的核心功能，需要单独处理：
- MVP 阶段使用 ts-fsrs 默认参数
- 待用户攒够 ≥1000 次复习，提供"重新优化参数"按钮
- 实现方案：纯 TS optimizer 实现（基于 gradient descent）或 WASM 包装 fsrs-rs
- 这是 P2 工作，不阻塞 MVP

## 七、调度器（Scheduler）

### 今日列表计算

```typescript
class Scheduler {
  constructor(private storage: StorageService, private fsrs: FSRSService) {}

  getTodayList(): NoteState[] {
    const now = new Date();
    const dayStart = this.getDayStart(now);
    const settings = this.storage.data.settings;

    const candidates = Object.values(this.storage.data.notes)
      .filter(n => !n.excluded)
      // 首次复习时间到了 + 未达到今日上限
      .filter(n => new Date(n.nextReview) <= now)
      // 今日未复习过（lastReview < dayStart）
      .filter(n => !n.lastReview || new Date(n.lastReview) < dayStart);

    // 计算当前 R 值，升序排序
    const withR = candidates.map(n => ({
      note: n,
      r: this.fsrs.calculateRetrievability(n, now),
    }));
    withR.sort((a, b) => a.r - b.r);

    // 取前 dailyLimit 条
    return withR.slice(0, settings.dailyLimit).map(x => x.note);
  }

  // 日边界计算
  getDayStart(at: Date): Date {
    const boundaryHour = this.storage.data.settings.dayBoundaryHour; // 4
    const d = new Date(at);
    d.setHours(boundaryHour, 0, 0, 0);
    if (d > at) d.setDate(d.getDate() - 1); // 凌晨 0-4 点属于昨天
    return d;
  }

  // 取下一条
  getNextUnreviewed(currentPath: string): NoteState | null {
    const list = this.getTodayList();
    const idx = list.findIndex(n => /*...需要额外机制标记当前复习进度...*/);
    return list[idx + 1] ?? null;
  }
}
```

### 实时计算的性能

假设 vault 有 5000 条笔记：
- 遍历：5000 次对象迭代
- R 值计算：5000 次幂函数求值
- 排序：5000 条排序 = ~60000 次比较（O(n log n)）

在现代 CPU 上全部加起来 **<5ms**。零感知。

如果未来扩展到 50000+ 条笔记，需要优化：
- 预先存储当前 nextReview，用索引筛选
- 按 nextReview 分桶
- 做日缓存（如前面讨论过的 thought）

MVP 不做。

### "今日进度"的状态标记

实时计算无法直接回答"这是今天的第 N 条"。需要一个 session 状态：
```typescript
class ReviewSession {
  private todayReviewed: Set<NotePath> = new Set(); // 今日已复习的 path
  private sessionStart: Date;

  completeCurrent(path: NotePath) {
    this.todayReviewed.add(path);
  }

  getProgress(): { done: number; total: number } {
    const list = scheduler.getTodayList();
    return {
      done: this.todayReviewed.size,
      total: this.todayReviewed.size + list.length,
    };
  }
}
```

这个 ReviewSession 只在内存，不持久化——过 04:00 自动清空。

## 八、编辑监听与 diff

### EditTracker 完整逻辑

```typescript
class EditTracker {
  private debounceTimers = new Map<NotePath, NodeJS.Timeout>();

  onModify(file: TFile) {
    const path = file.path;
    if (this.debounceTimers.has(path)) {
      clearTimeout(this.debounceTimers.get(path)!);
    }
    this.debounceTimers.set(path, setTimeout(() => {
      this.checkAndTrigger(file);
      this.debounceTimers.delete(path);
    }, 5000));
  }

  private async checkAndTrigger(file: TFile) {
    const path = file.path;
    const note = this.storage.data.notes[path];
    if (!note || note.excluded) return;

    const content = await this.plugin.app.vault.read(file);
    const newLength = content.length;
    const newHash = await hash(content);

    const lengthDiff = Math.abs(newLength - note.lastSnapshotLength);
    const ratioDiff = note.lastSnapshotLength === 0
      ? 1
      : lengthDiff / note.lastSnapshotLength;

    const settings = this.storage.data.settings;
    const exceedsAbsolute = lengthDiff > settings.editThresholdAbsolute;
    const exceedsRatio = ratioDiff > settings.editThresholdRatio;

    // 更新 snapshot（总是更新，即使不触发阈值）
    note.lastSnapshotLength = newLength;
    note.lastSnapshotHash = newHash;

    // 判断触发
    if (exceedsAbsolute && exceedsRatio) {
      // 7 天冷却期
      const lastTrigger = note.lastEditTriggerAt
        ? new Date(note.lastEditTriggerAt)
        : null;
      const now = new Date();
      const coolingDown = lastTrigger &&
        daysBetween(lastTrigger, now) < 7;

      if (!coolingDown) {
        const newState = this.fsrs.adjustStabilityForEdit(note);
        Object.assign(note, newState);
        note.lastEditTriggerAt = now.toISOString();
      }
    }

    await this.storage.save();
    this.notifyListChanged();  // 通知 SideBarView 刷新
  }
}
```

### Hash 实现

```typescript
async function hash(content: string): Promise<string> {
  const buf = new TextEncoder().encode(content);
  const digest = await crypto.subtle.digest('SHA-256', buf);
  return Array.from(new Uint8Array(digest))
    .map(b => b.toString(16).padStart(2, '0'))
    .join('')
    .slice(0, 16);  // 16 字符够用，省空间
}
```

## 十、错误处理与边缘情况

### 数据损坏

- `StorageService.load()` 捕获 JSON parse 错误
- 如果 data.json 损坏：备份原文件，使用空 data 启动，Notice 提示用户
- 不自动恢复——让用户感知到问题

### Obsidian 事件丢失

- 如果 Obsidian 因为某些原因（插件冲突、版本问题）没触发 rename 事件
- 导致 data.notes 里留下孤儿 path
- **清理机制**：每次启动时扫描 data.notes 的 key，验证对应文件是否存在
  - 不存在 → 等一段时间再次验证 → 仍不存在则归档

### 笔记路径含特殊字符

- 路径用 UTF-8 字符串，不做特殊处理
- data.json 的 key 对特殊字符安全

### FSRS 参数未训练时

- 使用 ts-fsrs 的 `generatorParameters()` 默认值
- 无需用户操作

### 首次安装无数据

- `storage.load()` 返回 null → 初始化默认 data
- 扫描 vault 所有现有笔记 → 批量 `onCreate`
- 所有现有笔记进入"等待首次复习"状态

## 十四、关键技术风险

### 风险 1：ts-fsrs API 变动

- ts-fsrs 仍在演进（当前对应 FSRS-6）
- 可能 FSRS-7 发布后 API 有变
- **缓解**：FSRSService 作为封装层，隔离 ts-fsrs 的 API 细节

### 风险 2：data.json 规模增长

- 重度用户一年可能积累 50K 条 revlog
- loadData() 每次启动全读
- **缓解**：M3 时考虑 revlog 分年度文件

### 风险 3：Obsidian 事件顺序不保证

- rename + modify 可能同时发生
- **缓解**：事件处理保持幂等；每次 modify 都检查 path 是否在 data 里

### 风险 4：复习 tab 和用户自己打开同一文件

- 用户自己在别的 tab 打开了相同笔记
- 复习 tab 里切换时可能混淆
- **缓解**：Obsidian 的 MarkdownView 本来支持多实例，编辑自动同步
